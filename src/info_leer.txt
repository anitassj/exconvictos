src/
Esta es la carpeta que contiene el código fuente del lado del servidor. 
Donde se maneja la lógica de la aplicación, las interacciones con 
la base de datos y las rutas.

controllers/: Los controladores son responsables de manejar las solicitudes 
            HTTP y la lógica. Aquí se define qué sucede cuando se accede a una 
            determinada ruta. Por ejemplo, controlador.js puede contener 
            funciones que gestionen las solicitudes de los usuarios, 
            como mostrar una página o procesar un formulario.

models/: Definen los modelos de datos, que representan las tablas de la
        base de datos en el código. Los modelos contienen la estructura de 
        los datos y cualquier lógica relacionada con ellos, 
        como validaciones o métodos para interactuar con la base de datos. 

routes/: Las rutas definen cómo se manejan las URL entrantes. Cada archivo 
        en esta carpeta normalmente corresponde a un conjunto de rutas 
        relacionadas. Por ejemplo, index.js maneja la ruta raíz (/), 
        mientras que login.js maneja las rutas relacionadas con el 
        inicio de sesión (/login).

app.js: es la entrada principal de la pág. Acá se configura el servidor web,
        se define middleware (como el manejo de errores o la autenticación) 
        y se enlazan las rutas definidas en src/routes/.
